1. Inroduction¶

Let's imagine that our body receive a signal, say through the eyes, this signal go through the eyes and flow from one group of neurons to the next until it reach a certain place where we make a certain judgement of that signal, say whether it is a cat or a flower. How to make a mathematical model of this?

Let $x$ be a signal. After receiving the signal, the input signal $x$ goes to a first group neurons $N_{1}^1,N_{2}^1,..,N_{k_1}^1$, neurons in the first layer, each $N_{i}^1$ processes information it received and passes it to deeper group of neuron $N_{1}^1,...,N_{k_2}^2$, neurons in the second layer, this process continue until signal reach a certain depth $L$ with neurons $N_{1}^L,...,N_{K_{L}}^L$, neurons in the $L $ layer, and the result from these last group combined to produce an output.

The simplest way to model that a neuron $N_{j}^i$ processing its inputs is to add all the inputs it receives to get a single number and then pass that number to the next neurons. However, just adding everything up is too restricted. To give extra freedom to the model and still keep it simple, we do weighted sum instead. Let's say neuron $N_{j}^i$ takes signal $x_1^{i-1},...,x_{k_{k-1}}^{i-1}$ from $N_1^{i-1},...,N_{k_{i-1}}^{i-1}$, respectively. The result is \begin{align*} x_{j}^i &= \sum_{l=1}^{k_i} a_{l}^i x_{l}^{i-1} + b_{ij} \\ &= (a_{i1}, \ldots, a_{ik_j}) \cdot (x_{(i-1)1}, \ldots, x_{(i-1)k_j}) + b_{ij} \end{align*} Here $(a_{1}^i,...,a_{k_j})^i$ is the weight vector of neuron $N_{j}^i$ uses to process information giving to it. Here each neuron has its own weight vectors. All this can be efficient pack as matrix multiplication and add a vector. Basically, the result from one layer of the neuron to the next is the result of an affine transforming of its input vectors.

The above implies that we can use affine transformations to model how each layer of neuron processes its input and passes that result to the next. So to model information flow in the brain through neuron, we can just compose such a map. However, composition of affine transform is an affine transform, this is equivalent to just one layers of neurons, to obtain the notion of multi-layers, we let the output of each layers passes through a non-linear function first before send it forward.

Definition

A neural network is a function obtained by composing affine linear functions with non-linear functions, generally known as activation functions. A neural network of $ L $ layer is a function $ NN: \mathbb{R}^n \to \mathbb{R}^m $ defined by $ NN(x) = Af_L \circ \sigma_L \circ \cdots \circ Af_2 \circ \sigma_2 \circ Af_1(x) $ where each $ Af_i: \mathbb{R}^{d_{i-1}} \to \mathbb{R}^{d_i} $ is an affine function, i.e., $ Af_i(x) = W_i x + b_i $, with $ W_i \in M_{d_{i-1}, d_i}(\mathbb{R}) $ and $ b_i \in \mathbb{R}^{d_i} $ and $d_0=m$ and $d_L=n$. Each $ \sigma_i $ is a non-linear activation function applied pointwise to the output of $Af_i $.

Example

Consider a three-layer neural network with the following architecture: The input layer, aka first layer, is two dimensional, the second layer has 2 neurons, the second layer has 3 neurons and the final output, aka 3 layer, has 3 dimension. Eah layers are defined by weight matrices and biases:

  • Layer 1 with $W_1 \in \mathbb{R}^{2 \times 2}$ a nd $b_1 \in \mathbb{R}^2$,

  • Layer 2 with $W_2 \in \mathbb{R}^{3 \times 2}$ and $b_2 \in \mathbb{R}^3$, and

  • Layer 3 with $W_3 \in \mathbb{R}^{3 \times 3}$ and $b_3 \in \mathbb{R}^3$.

The neural network $NN$ is expressed as: $ NN(x) = W_3 \sigma(W_2 \sigma(W_1 x + b_1) + b_2) + b_3 $.

If we define affine functions $Af_1(x) = W_1 x + b_1$, $Af_2(x) = W_2 x + b_2$, and $Af_3(x) = W_3 x + b_3$, then, $NN$can be rewritten as: $ NN(x) = Af_3(\sigma(Af_2(\sigma(Af_1(x)))))$

2. Universal Approximation¶

A neural network is defined by its weight matrices $W_i$ and bias vectors $b_i$, collectively known as parameters and denoted by $P = (W_1, W_2, \ldots, W_L, b_1, \ldots, b_L)$. These parameters can be thought long vector of dimension $K$ for some $K \in \mathbb{N}$, representing a point $P$ in $\mathbb{R}^K$. Each point corresponds uniquely to a neural network instance, denoted $NN_P$, which is structured according to the network's fixed architecture. This parameter-to-network mapping can be expressed as: $$ P \mapsto NN_P $$ The primary task of machine learning is to select the parameter $P$ that best approximates the function of interest. This point of view allow us to see ML is a techique of function approximation much like that of Fourier transform and Taylor expansion. One obvious question is what kind of function can be approximated by neural networks. The answer of this question coming from universal approximation theorems which we are now turn to in the next section.

Universal Approximation Theorem, G Cybenko The space $ N_2 = \bigcup_{N=1}^{\infty} \left\{ \sum_{i}^N\alpha_i \sigma(y_i^T x + b_i) \mid y_i \in \mathbb{R}^d \right\} $ is dense in $ \mathcal{C}(I_d, \mathbb{R}) $, where $ \sigma: \mathbb{R} \to \mathbb{R} $ is a fixed discriminatory function, to be defined below.

Definition We say that $\sigma$ is discriminatory if for a measure $\mu \in M(I_n)$, $$ \int_{I_n} \sigma(y^T x + \theta) \, d\mu(x) = 0 $$ for all $y \in \mathbb{R}^n$ and $\theta \in \mathbb{R}$ implies that $\mu = 0$.

Proof

Note that $ N_2 = \bigcup_{N=1}^{\infty} \left\{ \sum_{i}^N\alpha_i \sigma(y_i^T x + b_i) \mid y_i \in \mathbb{R}^d \right\} $ is a linear subspace of $\mathcal{C}(I_d,\mathbb{R})$. Assume it is not dense, then its closure $\bar{N_2}$ is proper Banach subspace of $\mathcal{C}$. We can define a linear functional $\ell:\bar{N_2}\to \mathbb{R}$ to be zero. Since $\bar{N_2}$ is a proper subspace, the Hanh-Banach's theorem allows us to extend $\ell$ to all of $\mathcal{C}$ such that $\ell(f)=0$ if $f\in \bar{N_2}$ and $\ell(f)\neq 0$ for some $f\notin \bar{N_2}$. In particular, there exist $f_0\in \mathcal{C}$ such that $\ell(f_0)\neq 0$.

Now follow Rieze-Markov-Kakutani representation theorem, we know there exist a positive Borel's measure $\mu$ on $I_d$ such that $$ \ell(f)=\int fd\mu $$ for all $f\in \mathcal{C}$. In particular $\int \sigma(y_i^Tx+b)d\mu=0$ for all $y\in \mathbb{R}^d$ and $b\in\mathbb{R}$. But $\sigma$ is discriminatory, this implies $\mu=0$. However, we know that $\int f_0d\mu=\ell(f_0)\neq 0$, this is a contradiction.

We can equip the space $\mathcal{C}(I_d,\mathbb{R})$ with the $L_p$ norm and there are similar results by Kurt Hornik in 1990.

Theorem The space $ N_2 = \bigcup_{N=1}^{\infty} \left\{ \sum_{i}^N\alpha_i \sigma(y_i^T x + b_i) \mid y_i \in \mathbb{R}^d \right\} $ is dense in the space of $\mathcal{C}(I_d,\mathbb{R})$ when $\mathcal{C}$ is equipped with the $L_p$ norm with $1\leq p\leq \infty$ and $\sigma$ is a continuous, bounded and non-constant function.

2. Implementation¶

Let's consider the task of finding a two-layer neural network to approximate $\cos(x)$ for $x \in [-10,10]$ with respect to the $L^2$ norm. Given $\varepsilon > 0$, the Universal Approximation Theorem tells us that there exists a neural network which is $\varepsilon$-close to $\cos(x)$ on this interval in the $L^2$ norm. The question then is, which neural network? Unfortunately, there is no straightforward way to identify it directly. What we can do is fix an architecture for the network and then search for the parameters $P$ that work best. We start with a random initial guess for $P$ and use gradient descent to iteratively improve this guess.

Let's set this mathematically. As an example, first, we select the architecture by setting the first layer to have dimension $13$ and the second layer to have dimension $5$. This means we are looking for a function $$ NN_P(x) = W_3(\sigma(W_2\sigma(W_1x+b_1)+b_2)+b_3) $$ where $W_1$ is a matrix that takes $\mathbb{R}^1$ to $\mathbb{R}^{13}$, $W_2$ is a matrix taking $\mathbb{R}^{13}$ to $\mathbb{R}^5$, $W_3$ is a matrix that takes $\mathbb{R}^5$ back to $\mathbb{R}$, and $b_1 \in \mathbb{R}^{13}, b_2 \in \mathbb{R}^5$ and $b_3 \in \mathbb{R}$. In this case, $P = (W_1, W_2, W_3, b_1, b_2, b_3)$ is a point in $\mathbb{R}^{13 \cdot 1 + 13 \cdot 5 + 5 \cdot 1 + 13 + 5 + 1=102}$.

Given $P \in \mathbb{R}^{102}$, we construct a function $NN_P(x)$ to approximate the cosine function. The error of this approximation is measured by the loss function, defined as: $$ \text{loss}(P) = \int_{-10}^{10} (\cos(x) - NN_P(x))^2 \, dx $$ The task is to optimize $\text{loss}: \mathbb{R}^{102} \to \mathbb{R}$. Of course we can't hope to find $P$ that truly is the minimum since the dimension is so large, what we can hope for is to find $P$ such that $\text{loss}(P)<\varepsilon$ for an acceptable $\varepsilon>0$.

Next, we need to be more realistic. We cannot compute $\text{loss}(P)$ directly because the function $NN_P(x)$ is difficult, if not impossible, to integrate analytically. To address this issue, we define a loss function at each point as: $$ l(P,x) = |\cos(x) - NN_P(x)|^2 $$ and then approximate the overall loss function $\text{loss}(P)$ by: $$ \text{loss}(P) \approx \text{loss}_D = \frac{\sum_{i=1}^N l(P,x_i)}{N} $$ where $D = \{x_1, \ldots, x_N\}$ is a set of points sampled from $[-10, 10]$. Here, $D$ indicates that evaluating the loss function depends on the data set $D$. Since this loss function is based on the data, it is called the empirical loss. Note that as $N \to \infty$, by continuity, $\text{loss}_D(P)\to 20\text{loss}(P)$.

Let's summarize where we are now. We have a dataset $D$. For each data point $x_i \in [-10,10]$, we can evaluate the point-wise loss for each parameter set $P$. Averaging these, we obtain the empirical loss: $$ \text{loss}_D(P) = \frac{\sum_{i=1}^N |\cos(x_i) - NN_P(x_i)|^2}{N} $$ While this empirical loss is computable, it remains challenging to optimize using critical points method. A method commonly used for this optimization is called gradient descent.

3. Gradient Decent Algorithm¶

Let's remind ourselves what gradient descent is. Consider $f: \mathbb{R}^d \to \mathbb{R}$. The goal is to find a point $P$ where $f(P)$ is the global minimum of $f$. The gradient descent algorithm suggests the following steps:

  1. Start at a point $P_0$.
  2. Compute the gradient $\nabla f(P_0)$.
  3. Take a small step $\eta$ in the direction opposite to the gradient, i.e., $P_1 = P_0 - \eta \nabla f(P_0)$.
  4. Repeat the above process.

$\eta$ is known as the learning rate. The reason we move in the direction opposite to the gradient is that the function decreases fastest in this direction.

We now state and prove that if $f$ is a nice function, the gradient decent algorithm will converge to a minimum. Note that most function we encounter are far from being nice.

Theorem Let $f:\mathbb{R}^d\to \mathbb{R}$ be a strongly convex, non-constant differentiable function where the derivative $Df:\mathbb{R}^d\to \mathbb{R}^d$ of $f$ is Lipschitz. Then $f$ has a unique minimum at $P_c$. Moreover, for $\eta$ sufficiently small, the sequence $$ P_{n+1}=P_n-\eta \nabla f(P_n) $$ converges to $P_c$ for any initial point $P_0$.

Definition Let $f:\mathbb{R}^d\to \mathbb{R}$. We say $f$ is convex if for all $P,Q\in \mathbb{R}^d$ we have $$ f(\lambda P+(1-\lambda Q)\leq \lambda f(P)+(1-\lambda)f(Q), $$ which implies $$f(P)-f(Q)\geq \nabla f(Q)^T(P-Q)$$ if $f$ is differentiable. A function $f$ is strongly convex if there exist $H>0$ such that $$ f(P)\leq f(Q)+\nabla f(Q)^T(P-Q)+H/2\|P-Q\|^2. $$ And $F:\mathbb{R}^d\to \mathbb{R}^d$ is Lipschitz if there exist $L>0$ such that $$ \|F(P)-\nabla F\|\leq L\|P-Q\| $$ for any $P,Q\in \mathbb{R}^d$.

Note: If $f$ is convex, with $Df$ being Lipschitz doesn't imply uniqueness of a minimum. $f(x,y)=x^2$ is an example of such function.

Proof Now we prove the convergence of the gradient descent sequence. Recall that the gradient descent update rule is given by: $$ P_{k+1} = P_k - \eta \nabla f(P_k) $$ where $ \eta $ is the step size (learning rate). We aim to show that this sequence $ P_k $ converges to the global minimum.

Since $ f $ is strongly convex, there exists a constant $ \mu > 0 $ such that for all $ P $ and $ Q $, $$ f(Q) \geq f(P) + \nabla f(P)^T (Q - P) + \frac{\mu}{2} \| P - Q \|^2. $$

Using the Lipschitz continuity of $ \nabla f $, we also have that there exists $ L > 0 $ such that: $$ \| \nabla f(P) - \nabla f(Q) \| \leq L \| P - Q \|. $$

Now, consider the difference in function values at consecutive points in the gradient descent sequence: $$ f(P_{k+1}) \leq f(P_k) + \nabla f(P_k)^T (P_{k+1} - P_k) + \frac{L}{2} \| P_{k+1} - P_k \|^2. $$

Substitute $ P_{k+1} = P_k - \eta \nabla f(P_k) $ into this inequality: $$ f(P_{k+1}) \leq f(P_k) - \eta \| \nabla f(P_k) \|^2 + \frac{L \eta^2}{2} \| \nabla f(P_k) \|^2. $$ This simplifies to: $$ f(P_{k+1}) \leq f(P_k) - \left( \eta - \frac{L \eta^2}{2} \right) \| \nabla f(P_k) \|^2. $$ For convergence, we require $ \eta $ to satisfy $ \eta < \frac{2}{L} $. In this case, the term $ \eta - \frac{L \eta^2}{2} $ is positive, and thus $ f(P_{k+1}) \leq f(P_k) $, implying that the function values decrease with each iteration of gradient descent.

Since $ f(P) $ is bounded below (by the global minimum), and the function values decrease monotonically, the sequence $ f(P_k) $ converges. Furthermore, because $ f $ is strongly convex, $ P_k $ converges to the unique global minimum.

This completes the proof of convergence of gradient descent to the global minimum.

3.1 Stochastic Gradient Decent¶

Consider the problem of training a neural network $NN(x)$ to learn a function $F:\mathbb{R}\to \mathbb{R}$. We don't know $F$ but we have $N$ samples of it, ie the data of $F$. Let $D=\{(x_1,y_1),(x_2,y_2),...,(x_N,y_N)\}$ be the data we collect about $F$.

Let's say we fix a neural network architecture, in other word, we fix the number of layers and the number of neuron the neural network has. This specification dictates the size of parameter $P$. Let's say $P\in \mathbb{R}^K$. Then the empirical loss function \begin{align*} \text{loss}_D(P) &= \dfrac{\sum_{i=1}^N |F(x_i) - NN_P(x_i)|^2}{N} \\ &= \dfrac{\sum_{i=1}^N |y_i - NN_P(x_i)|^2}{N} \end{align*} This is a function from $\mathbb{R}^K\to \mathbb{R}$. Training the neural network means using the gradient decent SGD to update the parameter $P$. Now if $N$ is large, the loss function for the whole data set can be computationally costly and the network can overfit the data. Stochastic gradient decent is proposed to solve this issue. Instead of computing the loss function for all $N$ data point, we instead compute it on subset of size $n$ where $n$ is much smaller than $N$.

Now fix $n<N$, the stochastic gradient decent algorithm:

  1. Select random $n$ data points from $D$, this make $S_n(1)\subset D$.
  2. Form loss function with respect to $S_n$, $\text{loss}_{S_n}(P)$.
  3. Use gradient decent to update $P$.
  4. Select new $n$ data points from $D$ randomly, call it $S_n$ again.
  5. Form $\text{loss}_{S_n}(P)$.
  6. Update $P$ using gradient decent.
  7. Repeat the above step.

The number $n$ used to compute the stochastic loss function is call batch size. Let's suppose we have $N=1000$ data points, if we make batch size $n=20$, then goes through all data points we must run the SGD $1000/20=50$ times. When we run through all the data, people say it is an epoch.

Remark: An interesting question is how do we know that evolution of $P$ via SGD algorithm converge to a stable position, let alone a global minimum.

Loss Functions for Classification Task: Cross-Entropy¶

Setup

We have $D=\bigcup_{i=1}^ND_i$ as disjoint union. The function $F:D\to \mathbb{R}^N$ defined by $F(x)=e_i$ if $x\in D_i$, where $e_1,e_2,...,e_N$ are the standard unit vectors, allow us to classify point in $D$ based on which subset $D_i$ they are in. For each $x$, the vector value $F(x)$ can be thought as a probability distribution on $\{D_1,D_2,...,D_N\}$. To use neural network to do clasification task mean to find a neural network $NN:D\to \mathbb{R}^N$ that approximate $F$ on $D$. The output $NN(x)\in \mathbb{R}^N$ should be thought of as a probability distribution.

Now need a way to compare to probability distribution.

Definition

Let's $P,Q:X\to [0,1]$ be a probability distribution over $X$. The cross entropy $H(P,Q)$ is defined as $ H(P,Q)=-\sum_{x\in X}P(X)\log(Q(x)) $

It would be cool if the cross entropy is a metric. However, it is not, one obvious thing is that it is not symmetric, ie $H(P,Q)\neq H(P,Q)$. What make the cross entropy a good candidate as a measure how close $Q$ is to $P$ is the following ineqaulity $ H(P,Q)\geq H(P,P) $ This means that the small we can make the cross entropy, the better we make $Q$ close to $P$.

Proof Note that $$ H(P, Q) = -\sum_{x \in X} P(x) \log Q(x) $$ can be rewritten as $$ H(P, Q) = -\sum_{x \in X} \left( P(x) \log \frac{Q(x)}{P(x)} + P(x) \log P(x) \right) $$ which simplifies to $$ H(P, Q) = H(P, P) - \sum_{x \in X} P(x) \log \frac{Q(x)}{P(x)} $$ Now we need to prove that $D_{KL}(P \| Q) = -\sum_{x \in X} P(x) \log \frac{P(x)}{Q(x)} \geq 0$. Note that $D_{KL}(P \| Q)$ is called the Kullback-Leibler (KL) divergence. It is $0$ if and only if $P = Q$.

Now, $\log: \mathbb{R} \to \mathbb{R}$ is a concave function, therefore it satisfies Jensen's inequality: $$ \sum_{i=1}^n \alpha_i \log(x_i) \leq \log\left( \sum_{i=1}^n \alpha_i x_i \right) $$ whenever $x_i \in [0, \infty]$ and $\alpha_i \in [0, 1]$ satisfies $\sum_{i=1}^n \alpha_i = 1$.

Applying this to the case where $\alpha_i = P(x_i)$ and $x_i = \frac{Q(x_i)}{P(x_i)}$, we get: $$ D_{KL}(P \| Q) \leq \log\left( \sum_{x \in X} P(x) \frac{Q(x)}{P(x)} \right) = \log(1) = 0 $$ Thus, $D_{KL}(P \| Q) \geq 0$.

Back to using neural network $NN$ to approximate distribution function. We know that the output of the neural network is a vector $NN(x)=(s_1,s_2,...,s_N)$ in $\mathbb{R}^N$. This is not any probability distribution. To deal with this problem, we pass the result of the neural network into softmax function defined as:

Definition The function $\text{SoftMax} : \mathbb{R}^n \to I^n$ is defined by $$ \text{SoftMax}(x_1, \ldots, x_n) = (p_1, \ldots, p_n) = \left( \frac{e^{x_1}}{\sum_{i=1}^n e^{x_i}}, \ldots, \frac{e^{x_n}}{\sum_{i=1}^n e^{x_i}} \right) $$

Recurrent Nueral Network¶

Imagine we have a collection of news articles $\mathcal{C}=\{A_1,A_2,...,A_K\}$ and we want to classify them into $L$={sport, politics, health}. Each news article is a string of words. To process these article need a way to represent each words numerically so that we can use mathematical techniques to do the classification task.

Assume $A_i=v_1...v_L$ where $v_{k}$ are words used to composed $A_i$ and $L$ is the length of $A_i$. Let $\mathcal{W}=\{w_1,w_2,...,w_N\}$ be the set of vocabulary used to composed $\{A_1,...,A_K\}$, ordered in some manner. In the palance of natural language processing, $\mathcal{C}$ is refereed to as corpus and $\mathcal{W}$ is the dictionary.

The goal is to represent $w_i$ numerically, one naive way to do this is to use number 1 to represent $W_1$, and so on. But the natural number has ordered structure, while we can't quite see that with words. So instead we should use vectors to represent words. More precisely, we want to find an embedding $E:\mathcal{W}\to \mathbb{R}^d$ for some $d$. One way to do this is to take $d=|\mathcal{W}|=N$ and send $w_i$ to the standard unit vector $e_i\in \mathbb{R}^N$, this is known as the one hot encoding. However, there are several issue with this method. First, the embedding dimension is high, for example there are around 600000 words in English. Second, $\mathbb{R}^N$ is a metric space, we would hope that that words $w_i$ and $w_j$ that are similar in meaning should be closer compare to those that are not, but every pair of different words has distance $\sqrt{2}$ in the one-hot-encoding.

Continuous word embedding deal with the issues mentioned about the one one encoding. There is $d\ll |\mathcal{W}|$ and a map $E:\mathcal{W} \to \mathbb{R}^d$ that with certain metric allow one to encoder each word as a vector.

The classification problem mention in the beginning now can be seen as classification of sequences of various length where each terms of the sequences are in $\mathbb{R}^n$. A more general set up is having finite sequence $x_1,x_2,...,x_L$ in $\mathbb{R}^d$, find a map $f$ to the output space $Y$, which can be continuous or discreet. This is an example of time series analysis. To deal with time series, we introduce recurrent neural network.

Recurrent Neural Network (RNN)¶

The text $A_1=x_1,x_2,...,x_{L_{A_1}}$, where $x_i\in \mathbb{R}^d$. Any model that can classify this text should look at the whole text to do so. One way to we can think of is to train a several layers neural network that take the concatenation of $x_i$ to get a long vectors. However, here we run to the same problem of large dimensional space. The recurrent neural network was proposed to process sequence of any length and at the same time require relatively small number of parameters.

A recurrent neural network (RNN) processing the sequence $x_1,x_2,...,x_T$ is defined as follow $$ \begin{cases} h_t=f(x_t,h_{t-1})\\ y_t=g(h_t) \end{cases} $$ for all $1\leq t\leq T$, with $h_t\in \mathbb{R}^H$ for some $H$ and $h_0=0$. We should think of $h_t$ as the memory of the system up to time $t$ and $y_t$ as the output. What happen is before anything happen, $h_0=0$, the system contain no information, and at time $t=1$, the system look at the first word $x_1$ and produce a memory. Next, it look at the second word $x_2$ and use previous memory $h_1$ to produce new memory $h_2$. The process repeat until the end of the sequence. In the classification task we mention earlier, we would want to use $y_T$ to make prediction of the class the news article belong to. $h_T$ obtained by the system look at all words that make up of $A_1$.

We think of $y_t$ as the output of the network. And $h_t$ are known as hidden vector and $H$ is called the hidden dimension.

To truely get the name neural network, $f$ and $g$ should be composition of affine maps and non-linear activation functions. One choice is the following $$ \begin{cases} f(x,h)=\sigma(W_xx+W_hh+b_h)\\ g(h)=W_yh+b_y \end{cases} $$ Now we present a version of the universal approximation theorem of the RNN.

Theorem Let $F_1(x_1),F_2(x_1,x_2),...,F_T(x_1,..,x_T)$ be continuous functions with $x_i\in A\subset \mathbb{R}^d$ and their output are in $\mathbb{R}^m$. If $A$ is compact, then for any $\varepsilon>0$, there exist appropriate weight $W_x,W_h,W_y,b_h,b_y$ such that $$ \sup_{1\leq t\leq T}\sup_{x_i\in A}|F_t(x_1,..,x_t)-y_t|\leq \varepsilon $$ where $y_t$ is the output of the recurrent neural network with weights $W_x,W_h,W_y,b_h$ and $ b_y$.

We will not prove this theorem here.

Other Variant of RNN¶

During training an RNN, we need to approximate gradient of the derivative of the model, which is a composition of functions. The longer the input sequence, the deeper the composition, and this leads to error in the approximation. Two phenomenons at least can happen, one is the gradient become close to zero, leading to slow training, and the other is the gradient is too large leading to unstable learning. To address these issue, various varient of RNN has been introduced. We will introduce two of them here and from now on we will call all of them as RNN.

Long Short Term Memory: LSTM¶

An LSTM cell consists of several key components called gates and cells. These components include the forget gate, input gate, cell state update, and output gate.

Given the input at time step $ t $, $ \mathbf{x}_t $, the previous hidden state $ \mathbf{h}_{t-1} $, and the previous cell state $ \mathbf{C}_{t-1} $, the LSTM cell performs the following operations:

  1. Forget Gate: $$ \mathbf{f}_t = \sigma(\mathbf{W}_f \mathbf{x}_t + \mathbf{U}_f \mathbf{h}_{t-1} + \mathbf{b}_f) $$ where $ \sigma $ is the sigmoid function.

  2. Input Gate: $$ \mathbf{i}_t = \sigma(\mathbf{W}_i \mathbf{x}_t + \mathbf{U}_i \mathbf{h}_{t-1} + \mathbf{b}_i) $$

  3. Cell State Update: $$ \mathbf{\tilde{C}}_t = \tanh(\mathbf{W}_C \mathbf{x}_t + \mathbf{U}_C \mathbf{h}_{t-1} + \mathbf{b}_C) $$ The new cell state is computed as: $$ \mathbf{C}_t = \mathbf{f}_t \odot \mathbf{C}_{t-1} + \mathbf{i}_t \odot \mathbf{\tilde{C}}_t $$ where $ \odot $ denotes element-wise multiplication.

  4. Output Gate: $$ \mathbf{o}_t = \sigma(\mathbf{W}_o \mathbf{x}_t + \mathbf{U}_o \mathbf{h}_{t-1} + \mathbf{b}_o) $$

  5. Hidden State Update: $$ \mathbf{h}_t = \mathbf{o}_t \odot \tanh(\mathbf{C}_t) $$

In summary, the equations for the LSTM cell are:

\begin{align*} \mathbf{f}_t &= \sigma(\mathbf{W}_f \mathbf{x}_t + \mathbf{U}_f \mathbf{h}_{t-1} + \mathbf{b}_f) \\ \mathbf{i}_t &= \sigma(\mathbf{W}_i \mathbf{x}_t + \mathbf{U}_i \mathbf{h}_{t-1} + \mathbf{b}_i) \\ \mathbf{\tilde{C}}_t &= \tanh(\mathbf{W}_C \mathbf{x}_t + \mathbf{U}_C \mathbf{h}_{t-1} + \mathbf{b}_C) \\ \mathbf{C}_t &= \mathbf{f}_t \odot \mathbf{C}_{t-1} + \mathbf{i}_t \odot \mathbf{\tilde{C}}_t \\ \mathbf{o}_t &= \sigma(\mathbf{W}_o \mathbf{x}_t + \mathbf{U}_o \mathbf{h}_{t-1} + \mathbf{b}_o) \\ \mathbf{h}_t &= \mathbf{o}_t \odot \tanh(\mathbf{C}_t) \end{align*}

Here, $ \mathbf{W}_f, \mathbf{W}_i, \mathbf{W}_C, \mathbf{W}_o $ are the weight matrices for the respective gates, $ \mathbf{U}_f, \mathbf{U}_i, \mathbf{U}_C, \mathbf{U}_o $ are the weight matrices for the hidden state, and $ \mathbf{b}_f, \mathbf{b}_i, \mathbf{b}_C, \mathbf{b}_o $ are the bias vectors.

Gated Recurrent Unit¶

A GRU cell consists of two main gates that manage the update of the hidden state: the reset gate and the update gate.

Given the input at time step $ t $, $ \mathbf{x}_t $, and the previous hidden state $ \mathbf{h}_{t-1} $, the GRU cell performs the following operations:

  1. Reset Gate: $$ \mathbf{r}_t = \sigma(\mathbf{W}_r \mathbf{x}_t + \mathbf{U}_r \mathbf{h}_{t-1} + \mathbf{b}_r) $$ where $ \sigma $ is the sigmoid function.

  2. Update Gate: $$ \mathbf{z}_t = \sigma(\mathbf{W}_z \mathbf{x}_t + \mathbf{U}_z \mathbf{h}_{t-1} + \mathbf{b}_z) $$

  3. Candidate Hidden State: $$ \mathbf{\tilde{h}}_t = \tanh(\mathbf{W}_h \mathbf{x}_t + \mathbf{U}_h (\mathbf{r}_t \odot \mathbf{h}_{t-1}) + \mathbf{b}_h) $$ where $ \odot $ denotes element-wise multiplication.

  4. Hidden State Update: The new hidden state is a linear interpolation between the previous hidden state and the candidate hidden state: $$ \mathbf{h}_t = (1 - \mathbf{z}_t) \odot \mathbf{h}_{t-1} + \mathbf{z}_t \odot \mathbf{\tilde{h}}_t $$

In summary, the equations for the GRU cell are:

\begin{align*} \mathbf{r}_t &= \sigma(\mathbf{W}_r \mathbf{x}_t + \mathbf{U}_r \mathbf{h}_{t-1} + \mathbf{b}_r) \\ \mathbf{z}_t &= \sigma(\mathbf{W}_z \mathbf{x}_t + \mathbf{U}_z \mathbf{h}_{t-1} + \mathbf{b}_z) \\ \mathbf{\tilde{h}}_t &= \tanh(\mathbf{W}_h \mathbf{x}_t + \mathbf{U}_h (\mathbf{r}_t \odot \mathbf{h}_{t-1}) + \mathbf{b}_h) \\ \mathbf{h}_t &= (1 - \mathbf{z}_t) \odot \mathbf{h}_{t-1} + \mathbf{z}_t \odot \mathbf{\tilde{h}}_t \end{align*}

Here, $ \mathbf{W}_r, \mathbf{W}_z, \mathbf{W}_h $ are the weight matrices for the respective gates, $ \mathbf{U}_r, \mathbf{U}_z, \mathbf{U}_h $ are the weight matrices for the hidden state, and $ \mathbf{b}_r, \mathbf{b}_z, \mathbf{b}_h $ are the bias vectors.

RNN Based Language Model¶

A language model is a function that takes sequence of words as input and output a new word. This allows one to get another new word using the word just genereated and the previous word and one can keep produce as many word as one want this way. This is called auto-regressive model.

In this section, we would build a character level language model based on RNN.

The strategy is as follow. First we one hot encode each character, ie if the list of characers are $\{c_1,c_2,..,c_K\}$, then $one_hot(c_j)=e_j\in \mathbb{R}^K$. Next we will project this embedding into another space $\mathbb{R}^d$ via a map $E:\mathbb{R}^K\to \mathbb{R}^d$, this process is called continuous embedding, and it can be done by multiplying $e_j$ by a matrix $E$, which is a learnable parameters. This whole process allow us to turn a sequnece of character to a sequence of vectors in $\mathbb{R}^d$, and from here we can use an RNN to approximate $$ P(w_{n+1}|w_1,...,w_n). $$ This conditional probablity is a map $P:(w_1,..,w_n)\to \mathbb{R}^K$.

class RNNLanguageModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.embd_table = nn.Embedding(vocab_size, n_embd)
        self.lstm = nn.LSTM(n_embd, hidden_size, num_layers=num_layer, batch_first=True)
        self.proj = nn.Linear(hidden_size, vocab_size)

    def forward(self, idx, targets=None):
        B, T = idx.shape  
        tok_emb = self.embd_table(idx)
        output, _ = self.lstm(tok_emb)
        logits = self.proj(output)
        
        if targets is None:  # Corrected variable name
            loss = None
        else:
            B, T, C = logits.shape
            logits = logits.view(B * T, C)
            targets = targets.view(B * T)
            loss = F.cross_entropy(logits, targets)
        
        return logits,loss

    def generate(self, idx, generate_length, seq_length):  # Added seq_length as argument
        hidden = None
        for _ in range(generate_length):
            idx_cond = idx[:, -seq_length:]  # Use seq_length argument
            tok_emb = self.embd_table(idx_cond)
            output, hidden = self.lstm(tok_emb, hidden)
            logits = self.proj(output[:, -1, :])
            probs = F.softmax(logits, dim=-1)
            idx_next = torch.multinomial(probs, num_samples=1)
            idx = torch.cat((idx, idx_next), dim=1)
        return idx
